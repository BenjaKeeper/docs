# Manual de Uso del Sistema MED360

---

## 1. Introducción

El sistema **MED360** es una solución integral diseñada para gestionar de manera segura y eficiente los procesos operativos de inventario, cargas de insumos, reposiciones y auditorías. Implementado para VIA Inversiones y respaldado por AWS, este sistema garantiza altos estándares de seguridad y adaptabilidad para distintos entornos hospitalarios o empresariales.

![Vista general del sistema MED360](https://ruta.de/tu/imagen_general.jpg)

## 2. Acceso y Autenticación

<AccordionGroup>
  <Accordion defaultOpen={true} icon="login" title="Ingreso al Sistema">
    El acceso a **MED360** se realiza a través de métodos de autenticación seguros:
    
    - **Autenticación con Usuario y Contraseña:**  
      Los usuarios deben ingresar sus credenciales asignadas para acceder al sistema.  
      
      ![Pantalla de login con usuario y contraseña](https://ruta.de/tu/imagen_login.jpg)
    
    - **Inicio de Sesión con Cuenta Google:**  
      Empresas con acceso a Gmail pueden ingresar de manera directa y simplificada mediante su cuenta Google.  
      
      ![Opción de inicio de sesión con Google](https://ruta.de/tu/imagen_google.jpg)
  </Accordion>

  <Accordion defaultOpen={true} icon="user" title="Registro y Asignación de Roles">
    - **Proceso de Registro:**  
      Se solicitan datos básicos como nombre, correo y contraseña. En el caso de utilizar Google, se autentica automáticamente la cuenta.
      
    - **Asignación de Roles:**  
      Tras el registro, se generan roles específicos (por ejemplo, administrador, usuario de pabellón) que determinan el nivel de acceso y las funciones disponibles en el sistema.  
      
      ![Proceso de asignación de roles y perfil de usuario](https://ruta.de/tu/imagen_roles.jpg)
  </Accordion>

## 3. Gestión de Inventario

<AccordionGroup>
  <Accordion defaultOpen={true} icon="inventory" title="Visualización del Inventario">
    La pantalla principal de inventario proporciona una vista completa del estado actual del stock.
    
    **Elementos Clave:**
    - Stock actual
    - Lead time (tiempo estimado de reposición)
    - Stock mínimo de seguridad
    - Fecha de reposición
    
    ![Pantalla principal de inventario](https://ruta.de/tu/imagen_inventario.jpg)
  </Accordion>

  <Accordion defaultOpen={true} icon="edit" title="Modificación y Registro de Cambios">
    - **Proceso de Retiro de Productos:**  
      Cuando se retira un producto (por ejemplo, por requerimiento del proveedor), se solicita al usuario que indique la razón (como “Retiro por proveedor”).
    
    - **Registro Detallado:**  
      Cada modificación se registra en el sistema (logs) para facilitar la trazabilidad y auditoría.  
      
      ![Ejemplo de log o registro de cambios](https://ruta.de/tu/imagen_log.jpg)
  </Accordion>

  <Accordion defaultOpen={true} icon="upload" title="Ingreso de Stock mediante Guías de Despacho">
    - **Creación de Guías de Despacho:**  
      Las guías permiten registrar la llegada de nuevos productos, replicando el proceso tradicional en papel.
    
    - **Métodos de Ingreso:**
      - **Uso de Pistola Lectora:**  
        Escaneo directo del código del producto para ingreso automático.
      - **Búsqueda Manual:**  
        En caso de no disponer del escáner, se puede buscar el producto mediante nombre o SKU.
    
    - **Validación y Guardado:**  
      El sistema valida la información (SKU, lote, UPN) y actualiza el inventario en tiempo real.  
      
      ![Ingreso de productos mediante guía de despacho](https://ruta.de/tu/imagen_guia.jpg)
  </Accordion>
</AccordionGroup>

## 4. Carga de Insumos

<AccordionGroup>
  <Accordion defaultOpen={true} icon="clipboard" title="Carga de Insumos">
    Esta sección permite registrar de manera detallada cada procedimiento en el que se utilizan productos o medicamentos.
    
    **Datos Requeridos:**
    - Fecha del procedimiento
    - Nombre del paciente
    - Área o subruta del procedimiento
    - Médico tratante
    - Prestaciones asociadas (configurables según el servicio, por ejemplo, Modinamia)
    
    **Proceso de Registro:**  
    Se puede ingresar el producto mediante escaneo o búsqueda manual, y asociarlo automáticamente al paciente.  
      
    ![Formulario de carga de insumos](https://ruta.de/tu/imagen_carga_insumos.jpg)
    
    **Generación de Reportes:**  
    Una vez guardado el registro, se genera un reporte detallado de costos y consumo de insumo para su posterior análisis.  
      
    ![Reporte de carga de insumo](https://ruta.de/tu/imagen_reporte.jpg)
  </Accordion>
</AccordionGroup>

## 5. Reposiciones

<AccordionGroup>
  <Accordion defaultOpen={true} icon="refresh" title="Reposiciones">
    La funcionalidad de reposiciones permite gestionar la solicitud y reposición de productos cuando se alcanza el stock mínimo.
    
    - **Visualización de Productos a Reponer:**  
      El sistema lista los productos que requieren reposición, destacando aquellos críticos.
      
    - **Solicitud al Proveedor:**  
      Se envía una notificación o solicitud directa al proveedor para la reposición del producto.
      
    ![Reposiciones](https://ruta.de/tu/imagen_reposiciones.jpg)
  </Accordion>
</AccordionGroup>

## 6. Gestión de Maestros

<AccordionGroup>
  <Accordion defaultOpen={true} icon="database" title="Maestro de Productos">
    - **Creación y Edición de Productos:**  
      Permite ingresar datos fundamentales como el nombre del insumo, grupo (licitación, consignado), medidas, costo neto y otros identificadores (CFN, UPN).  
      
      ![Ingreso de un nuevo producto](https://ruta.de/tu/imagen_nuevo_producto.jpg)
  </Accordion>

  <Accordion defaultOpen={true} icon="barcode" title="Maestro de CKUs">
    - **Generación de CKUs:**  
      Para productos que requieren seguimiento especial, se pueden generar CKUs (Códigos de Kilo Unidad) que facilitan la gestión y el control.
    
    - **Impresión y Exportación:**  
      Los CKUs pueden descargarse en formato imprimible para su uso físico.  
      
      ![Ejemplo de CKU generado](https://ruta.de/tu/imagen_cku.jpg)
  </Accordion>

  <Accordion defaultOpen={true} icon="user-group" title="Gestión de Proveedores">
    - **Registro de Proveedores:**  
      Se ingresan datos comerciales y de contacto, además del tiempo estimado de entrega (lifetime).  
      
      ![Gestión de proveedores](https://ruta.de/tu/imagen_proveedores.jpg)
  </Accordion>

  <Accordion defaultOpen={true} icon="user" title="Gestión de Profesionales y Clientes">
    - **Profesionales:**  
      Registro de roles y datos (médicos, enfermeros, etc.) para su asignación en la carga de insumos.
    
    - **Clientes:**  
      Configuración de facturación según el establecimiento (hospital, pabellón, etc.).  
      
      ![Perfiles de profesionales y clientes](https://ruta.de/tu/imagen_profesionales.jpg)
  </Accordion>

  <Accordion defaultOpen={true} icon="layers" title="Categorías y Licitaciones">
    - **Configuración de Insumos:**  
      Se asigna el nombre, categoría y precios de venta a cada insumo, permitiendo una clasificación precisa para la facturación y gestión interna.  
      
      ![Categorías y licitaciones](https://ruta.de/tu/imagen_categorias.jpg)
  </Accordion>
</AccordionGroup>

## 7. Reportes y Auditoría

<AccordionGroup>
  <Accordion defaultOpen={true} icon="history" title="Registro y Logs del Inventario">
    - **Historial de Modificaciones:**  
      Se almacena información detallada sobre cada cambio en el inventario, incluyendo quién realizó la modificación y cuándo.  
      
      ![Historial de cambios](https://ruta.de/tu/imagen_logs.jpg)
  </Accordion>

  <Accordion defaultOpen={true} icon="export" title="Exportación de Datos">
    - **Exportar a Excel:**  
      La mayoría de las tablas e informes permiten exportar los datos a Excel para análisis externo y auditorías.  
      
      ![Exportación a Excel](https://ruta.de/tu/imagen_excel.jpg)
  </Accordion>

  <Accordion defaultOpen={true} icon="report" title="Reportes de Prestaciones y Productos Implantables">
    - **Detalle del Procedimiento:**  
      Reportes que incluyen datos del paciente, producto, lote, modelo, proveedor y la guía de despacho asociada.  
      
      ![Reporte de prestaciones](https://ruta.de/tu/imagen_reporte_prestaciones.jpg)
  </Accordion>

  <Accordion defaultOpen={true} icon="check-circle" title="Auditoría de Inventario">
    - **Proceso de Auditoría:**  
      Se realiza un escaneo completo de los productos en un pabellón, comparando el inventario físico con el registrado en el sistema para identificar mermas o inconsistencias.
    
    - **Reporte de Auditoría:**  
      Al finalizar, se genera un informe detallado que incluye:
      
      - Cantidad inicial y final del inventario
      - Productos auditados y no encontrados
      - Registro de mermas detectadas
      
      ![Auditoría de inventario](https://ruta.de/tu/imagen_auditoria.jpg)
  </Accordion>
</AccordionGroup>
